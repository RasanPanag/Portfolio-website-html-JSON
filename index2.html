Criterias Performed on Project Management APP:

  • • Employees will be able to view project, milestone information and they can add comments on tasks and update the status.
  • • Employees will not be able to edit each other's tasks. But managers will be able to view all the tasks of the employees who report to him/her.
  • • There are different views set up for the employees. Testing team employees will see the task types specifically for their team. Task Type-Picklist Values (Functional Testing and Integration testing). Development teams should see specific task types for their team. Task Type-Picklist Values (Unit Testing, Coding and Functional Requirement).
  • • Both milestones and projects have paths.
  • • Stage-specific fields are available for the managers to view and edit on projects.
  • • On completion of each milestone Manager will be notified through email.
  • • Employees will get notification on each task assignment.
  • • Actual hours on the project will be rolled up from task through milestone
  • • There is Project health Indicator field that will show green when all the milestones are within the timeline. The health Indicator will turn red if any of the milestones breaches thetimeline
  • • Once a manager creates a project and it is in the proposal stage it will go to the Director for approval before submitting it to the client.Once approved the stage should be changed to Negotiation.
  • • The CEO of the company should receive reports about the project progress on the last day of each month.
  • • Project should go through the approval process before it is assigned to any resources.
  • • Employees can log time against task. Not more than 12 hours can be added for one day including all the time logs.
