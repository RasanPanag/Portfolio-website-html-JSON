Cricet Teams

This project will show you how we can enter into a deadlock while running a program on a terminal and it will also show how we can bypass the deadlock. There are two C programming files one file is based on the scenario that there are five cricket teams numbered zero to four and there are 4 sets of three wickets. Each team needs two sets to practice for their upcoming matches but, on the ground, each team has one set to use. So, for them, to practice they must share a set of wickets. The set on their right-hand side has the same number as the team. So, each team will reach its own set. When they have their assigned set of wickets. Then the team will reach wickets that are assigned to their neighbor. After one team has both sets of wickets then they will practice. After practice then that team can return the set of wickets to their original position on the ground. The process is repeated until there is no more team left to practice. White the other file is based on the same scenario but, it has four teams that will attempt to practice at the same time. The new implementation uses the semaphore "num_can_practice" to limit the number of teams who can practice at the same time. This semaphore is initialized to four, one less than the number of philosophers. Also, this file is created to solve the problem of actual or potential deadlocks.

Everything was created by using C programming-language on Linux Terminal

tic tac toe

This is a project in which I have created the concept of Tic Tac Toe by using the python language There will be two players in a game. Two signs represent each player. The general signs used in the game are "X" and "O". Finally, there will be a board with 9 boxes. First, one user (either "X" or "O") will place their sign in one of the available empty boxes. Next, the second user will place their sign in the remaining boxes The game will end in winning if one of the players has placed their respective signs three consecutive times in a straight line row-wise, column-wise, or diagonally. Or Else, it is a draw, if all boxes are filled.
